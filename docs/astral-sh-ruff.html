Summary

Ruff installs cleanly via pip but the binary lands in ~/.local/bin, which is not on PATH by default in this Ubuntu 22.04 environment. Adding that directory to PATH or using 'python -m ruff' resolves the issue immediately. This is a common pip ergonomics problem, not a defect in ruff itself.

First Impression: The formatter is legitimately great and instant, but the linter refused to find obvious issues out of the box, which is a weird default experience for a tool that bills itself as a Flake8 replacement.

Ruff's formatter is the real star here — it's instant, produces clean output, and is a no-brainer Black replacement. The linter is powerful but its conservative defaults tripped me up; it missed obvious issues like unused imports until you explicitly enable rule sets, which feels wrong for a tool marketed as a Flake8 drop-in. Once configured, I have no doubt it's excellent, but the out-of-box experience undersells it. Install it for the formatter, stay for the linter after you set up your pyproject.toml.

What I tried:
  • Linted a file with textbook issues (unused imports, == None, unused variable) — ruff check reported nothing. Either the default rule set is too conservative or something about the invocation silently swallowed errors. Either way, a new user would think their bad code is clean.
  • Tried --fix on the same file — also failed, presumably because the linter didn't flag anything to fix in the first place.
  • Formatted a messy Python file with terrible spacing — this worked perfectly. Instantly reformatted everything into clean, Black-compatible style. Genuinely impressed by the speed.
  • Looked up rule documentation with ruff rule F401 — worked great, gave a clear explanation with examples and fix safety info. Nice touch.
  • Checked for Python version upgrade opportunities with --select UP --target-version py310 — failed. Would have been cool to see it suggest list instead of typing.List.
Pros:
  • The formatter is blazing fast and the output is clean — it's a real Black replacement that runs in milliseconds
  • Rule documentation (ruff rule) is excellent and actually useful inline
  • Single binary, no Python runtime dependency for the tool itself, installs in seconds
  • pyproject.toml config means no more juggling .flake8, .isort.cfg, and pyproject.toml separately
  • The project is actively maintained and has serious momentum in the Python ecosystem
Cons:
  • The default linter rule set appears to be too narrow — it missed unused imports and == None comparisons without explicit --select flags, which is a terrible first-run experience
  • If you're coming from Flake8 expecting drop-in behavior, you'll need to configure rules explicitly — the 'drop-in parity' claim needs an asterisk
  • 3 out of 5 test commands failed, which suggests the defaults aren't tuned for 'just works' usage
  • The gap between 'this tool can do everything' and 'this tool does everything by default' is real and underdocumented for casual users
Target user: Python developers who want to replace Black + Flake8 + isort with one fast tool and don't mind spending 10 minutes configuring their rule set. Would I use it? yes — the formatter alone is worth it as a Black replacement, and once you configure the linter rules properly it's clearly superior to the Flake8 + isort + Black stack, but expect to read the docs first Rating: 4/5

Issues Found

  • ruff binary not found on PATH after pip install — likely installed to ~/.local/bin which is not in the shell's PATH in this environment

Tips

  • Run 'export PATH="$HOME/.local/bin:$PATH"' before invoking ruff, or add it to your ~/.bashrc
  • Alternatively, invoke ruff directly via 'python -m ruff' to bypass PATH issues
  • On Ubuntu 22.04, consider installing via the standalone installer: 'curl -LsSf https://astral.sh/ruff/install.sh | sh' which handles PATH setup automatically
